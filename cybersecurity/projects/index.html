Wpdiscuz-lab

WordPress wpDiscuz Python

Custom Python-based network scanner adapted to discover and enumerate WordPress instances and vulnerable plugins. Includes automated port mapping, banner grabbing and a companion exploitation lab for wpDiscuz vulnerability.

View code Lab PDF Vulnerability writeup
📝 Lab notes — key steps (click to expand)
  1. Recon & discovery — enumeration of hosts and WordPress instances.
  2. Detection of vulnerable plugin — fingerprinting wpDiscuz versions.
  3. Use of the exploit & webshell — upload / trigger RCE payload.
  4. Reverse shell — obtain interactive session on target.
  5. Privilege verification & sudo — identify user and sudo rights.
  6. Escalation with Nokogiri — exploit path for local privilege escalation.
  7. Reading sensitive files — extraction of config and credentials.
  8. Using pspy64 — process monitoring to catch cron tasks and secrets.
  9. Exploitation via Wildcard Injection — secondary vector discovery.
  10. Keys and root access — retrieving keys and achieving root.
  11. Evidence (flags) — captured artefacts and PoC evidence.
  12. Mitigations & recommendations — patching, least privilege, input validation, WAF rules.
